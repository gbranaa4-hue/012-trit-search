# 012 Ternary Triadic Networks

A complete computing stack built on ternary logic ({-1, 0, +1}) and
triadic neural architecture (Observer / Shadow / Light).

## Structure

```
012-ternary/
├── experiments.py          # Full benchmark: CIFAR-10 + STL-10 + ablation
├── hardware/
│   ├── trit_pkg.sv         # Ternary type definitions
│   ├── consensus_gate.sv   # 012 native gate (3-input majority vote)
│   ├── trit_not.sv         # Ternary NOT
│   ├── trit_add.sv         # Ternary adder with carry
│   ├── trit_mac.sv         # Multiply-accumulate (no multipliers)
│   ├── trit_register.sv    # Ternary D flip-flop
│   ├── triadic_pe.sv       # Triadic processing element
│   ├── 012_top.sv          # System top level
│   ├── testbench.sv        # 36-case hardware validation (36/36 pass)
│   ├── vivado_synth.tcl    # Synthesis script (Xilinx Ultrascale+)
│   └── ternary_rtl.py      # Python validation of hardware logic
├── paper/
│   └── 012_paper.md        # Full paper draft
└── results/
    └── 012_full_benchmark.json   # Benchmark data (generated by experiments.py)
```

## Key Results

| Metric                    | Value                        |
|---------------------------|------------------------------|
| Parameters                | 396,174 (28.2x fewer than ResNet18) |
| Ternary model size        | 0.078 MB (20.2x compression) |
| CIFAR-10 stability gain   | +8.17pp vs parameter-matched CNN |
| STL-10 stability gain     | +11.55pp vs parameter-matched CNN |
| Hardware tests            | 36/36 pass                   |
| Est. FPGA power           | ~0.8W (vs ~300W GPU)         |

## Run

```bash
# Software experiments (requires GPU, ~40 min on T4)
python experiments.py

# Hardware validation (requires iverilog)
iverilog -g2012 -o sim hardware/trit_pkg.sv hardware/trit_register.sv \
    hardware/trit_not.sv hardware/trit_add.sv \
    hardware/consensus_gate.sv hardware/testbench.sv
vvp sim

# FPGA synthesis (requires Vivado)
vivado -mode batch -source hardware/vivado_synth.tcl
```
